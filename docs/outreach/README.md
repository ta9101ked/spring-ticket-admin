# 営業・提案キット（Phase 5）

学習用ポートフォリオ [spring-ticket-admin](https://github.com/ta9101ked/spring-ticket-admin) を使った案件獲得用メモ。  
**実応募の送信は自分で行う。** ここでは文面とルールを固定する。

## 1. 売り文句（一文）

> Java / Spring（および PHP/Laravel の業務画面）で、社内ツール・管理画面の小改修を **8〜20時間単位** で請けます。要件の整理から実装・動作確認まで一気通貫できます。

補助線（差別化・必要なら）：

> ナイトワーク店舗の業務（日払い・シフト・来店管理など）にも現場理解があります。Webに詳しくない事業者との要件整理が得意です。

## 2. 単価ルール（崩すな）

| 区分 | 時間単価 | 条件 |
|---|---|---|
| 通常提示 | **4,000円** | 基本これで出す |
| 下限 | **3,500円** | これ未満は原則お断り |
| 例外帯 | **3,000〜3,500円** | 実績公開OK + 完了定義あり + 継続/紹介の見込みがある時だけ |
| 3ヶ月目標 | **加重平均で5,000円超** | 全案件いきなりではない |

稼働制約（自分用）：

- 狙い：週10時間 / 上限：週15時間
- 目安帯：平日 14:00–16:00
- 粒度：完了まで 8〜20時間の改修1本

## 3. プロフィール用・短い自己紹介

Java / Spring Boot で業務システムの改修・保守経験があります。  
直近は PHP/Laravel の業務画面（店舗運営系）の設計・実装にも取り組み、一覧・検索・登録・状態管理といった管理画面の型を自分で説明できるサンプルを公開しています。

ブランク期間は「何もしていない」ではなく、**個人開発と設計力の再構築**に使いました。  
最初の案件は小さな改修から確実に納品し、継続につなげたいです。

GitHub: https://github.com/ta9101ked/spring-ticket-admin

## 4. 提案テンプレ（コピペ用・毎回1行カスタム）

件名例: `【管理画面の小改修】Spring/PHP で8〜20h単位の改修が可能です`

本文:

```text
はじめまして。池田と申します。

募集内容を拝見し、〇〇（※ここだけ案件ごとに書く：例「一覧への検索追加」「ステータス項目の追加」）に対応できそうと感じご連絡しました。

■対応できること
- 業務システム / 管理画面の小改修（追加・修正・不具合対応）
- Java/Spring、PHP/Laravel
- 完了定義を切った 8〜20時間程度の粒度での請負

■根拠
- Spring Boot のチケット管理サンプル（CRUD・検索・バリデーション・テスト・設計書）を公開しています
  https://github.com/ta9101ked/spring-ticket-admin
- 過去に業務システムの改修・保守、および設計への関与経験があります

■稼働
- 週10時間前後（上限15時間）
- 平日日中の一部時間帯を中心

■単価の目安
- 時間単価 4,000円前後（内容により相談）

要件の確認ポイントを1〜2点伺ってもよろしいでしょうか。
（例: 既存環境、完了条件、希望納期）

よろしくお願いいたします。
```
